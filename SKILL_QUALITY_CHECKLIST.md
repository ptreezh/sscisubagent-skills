# 技能质量审核清单

## 审核目的
确保所有技能设计符合 agentskills.io 标准，提供精心、全面、系统的思考和设计，实现渐进式信息披露和定性定量分离原则。

## 审核标准

### 1. 完整性 (Completeness) - 30分
- [ ] 技能文档包含所有必需部分 (10分)
  - [ ] 概述 (Overview)
  - [ ] 使用时机 (When to Use)
  - [ ] 快速开始 (Quick Start)
  - [ ] 核心流程 (Core Process)
  - [ ] 输出格式 (Output Format)
  - [ ] 质量标准 (Quality Standards)
  - [ ] 深入学习 (Advanced Learning)
  - [ ] 完成标志 (Completion Criteria)
- [ ] 每个部分都有实质性内容 (10分)
- [ ] 提供了充分的示例和案例 (10分)

### 2. 深度 (Depth) - 25分
- [ ] 理论基础充分 (10分)
- [ ] 实践指导详细 (10分)
- [ ] 提供了深入的学习资源 (5分)

### 3. 实用性 (Practicality) - 25分
- [ ] 提供了自动化工具 (10分)
- [ ] 操作步骤清晰可行 (10分)
- [ ] 包含质量控制机制 (5分)

### 4. 一致性 (Consistency) - 10分
- [ ] 遵循统一的结构标准 (5分)
- [ ] 术语使用一致 (5分)

### 5. 可维护性 (Maintainability) - 10分
- [ ] 包含测试用例 (5分)
- [ ] 提供了清晰的文档 (5分)

## 具体审核项目

### A. 基础元数据
- [ ] name: 技能名称清晰明确，符合命名规范
- [ ] description: 功能描述简洁准确，包含关键词
- [ ] version: 版本号遵循语义化版本规范
- [ ] author: 作者信息完整
- [ ] tags: 包含相关标签便于发现
- [ ] license: 许可证信息明确
- [ ] compatibility: 兼容性信息完整
- [ ] metadata: 扩展元数据完整（domain, methodology, complexity等）
- [ ] allowed-tools: 允许使用的工具列表明确

### B. 渐进式信息披露架构
- [ ] Level 1 元数据层 (~100 tokens, 始终加载) - 完整
- [ ] Level 2 指令层 (<5000 tokens, 触发时加载) - 完整
- [ ] Level 3 资源层 (按需加载, 不占用上下文) - 结构清晰
- [ ] 主控文档与参考文档分离 - 已实现

### C. 定性定量分离原则
- [ ] 定性分析任务由AI处理 - 已明确
- [ ] 定量计算任务由脚本处理 - 已实现
- [ ] 提示词文件用于定性指导 - 已创建
- [ ] Python脚本用于定量计算 - 已实现

### D. 技能文档结构
- [ ] 概述部分包含技能目的和意义
- [ ] 使用时机部分明确触发条件
- [ ] 快速开始部分提供简单使用方法
- [ ] 核心流程部分详细分步指导
- [ ] 输出格式部分规范标准化输出
- [ ] 质量标准部分包含检查清单
- [ ] 深入学习部分提供扩展资源
- [ ] 完成标志部分明确交付标准

### E. 工具支持

- [ ] 脚本包含类型注解和文档字符串

### F. 参考资料

### G. 测试覆盖

- [ ] 测试覆盖主要功能
- [ ] 包含性能基准测试

### H. 依赖管理
- [ ] 依赖关系明确声明
- [ ] 高级功能与降级实现并存

- [ ] 使用中文术语和概念

## 审核流程

### 第一步：自评
- 技能开发者根据此清单进行自评
- 对每个项目打分并记录

### 第二步：同行评审
- 由另一位开发者进行独立审核
- 验证自评结果

### 第三步：专家评审
- 由领域专家进行最终审核
- 确保理论和实践的准确性

### 第四步：系统验证
- 验证技能与agentskills.io标准的兼容性
- 测试技能的实际功能和性能

### 第五步：修订
- 根据审核反馈进行修订
- 重新进行审核直至达标

## 评分标准

- 优秀: 90-100分 (完全符合标准，可直接发布)
- 良好: 80-89分 (基本符合标准，少量改进后可发布)
- 合格: 70-79分 (符合基本要求，需要显著改进)
- 不合格: <70分 (不符合标准，需要重大改进)
