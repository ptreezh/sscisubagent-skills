# Paradigm模型质量标准

> **前置阅读**：建议先阅读 `six-components.md` 了解Paradigm结构。

## Strauss & Corbin的质量标准

基于 Strauss & Corbin (1998, pp.269-272) 的理论评估标准，Paradigm模型应满足以下四个维度：

---

## 1. 理论的完整性（Completeness）

### 标准

**所有六个组件都有充分的数据支持**

✅ **检查清单**：
- [ ] 现象：明确且有充分证据
- [ ] 因果条件：至少2个，有数据支持
- [ ] 语境：清晰描述情境
- [ ] 中介条件：至少2个，有数据支持
- [ ] 行动策略：至少3个，有数据支持
- [ ] 结果：短期和长期结果都有

**组件间的逻辑关系清晰**

✅ **检查清单**：
- [ ] 因果条件 → 现象：逻辑清晰
- [ ] 现象 → 行动策略：因果明确
- [ ] 行动策略 → 结果：关系清晰
- [ ] 中介条件影响策略效果

**模型能够解释大部分数据**

✅ **检查清单**：
- [ ] 覆盖率：≥80%的数据能被模型解释
- [ ] 反例：少于20%的反例
- [ ] 例外：能够解释例外情况

### 评估方法

**使用工具检查**：
```bash
python scripts/construct_paradigm.py \
  --input relationships.json \
  --output paradigm.json \
  --validate
```

查看 `validation.completeness_score`：
- ≥0.8：完整性良好
- 0.6-0.8：基本完整，可以改进
- <0.6：不完整，需要补充

**人工检查**：
- 逐一检查六个组件
- 回到原始数据验证
- 询问"是否还有重要的遗漏？"

### 案例

**完整性良好的Paradigm**：
```json
{
  "completeness_score": 0.85,
  "components": {
    "phenomenon": "学业适应困难",
    "causal_conditions": ["学习方法差异", "课程难度提升"],
    "context": ["重点大学环境", "远离家乡"],
    "intervening_conditions": ["个人性格", "社会支持", "学校资源"],
    "action_strategies": ["寻求学业支持", "调整学习方法", "建立社会网络"],
    "consequences": ["成绩提升", "自信增强", "过度依赖"]
  },
  "missing": []
}
```

**不完整的Paradigm**：
```json
{
  "completeness_score": 0.50,
  "components": {
    "phenomenon": "学业适应困难",
    "causal_conditions": ["学习方法差异"],
    "context": [],  // 缺失
    "intervening_conditions": [],  // 缺失
    "action_strategies": ["寻求帮助"],
    "consequences": []  // 缺失
  },
  "missing": ["context", "intervening_conditions", "consequences"]
}
```

## 2. 理论的密度（Density）

### 标准

**范畴之间有丰富的关系**

✅ **检查清单**：
- [ ] 关系数量：≥10个主要关系
- [ ] 关系类型：包含因果、条件、策略、互动
- [ ] 关系网络：形成连贯的网络

**关系有充分的证据支持**

✅ **检查清单**：
- [ ] 每个关系至少3个数据片段
- [ ] 关系强度≥0.5
- [ ] 关系在不同受访者中都出现

**理论有足够的复杂性**

✅ **检查清单**：
- [ ] 不是简单的线性因果
- [ ] 考虑了中介条件
- [ ] 识别了互动关系和反馈循环

### 评估方法

**使用工具检查**：
```bash
python scripts/build_relationships.py \
  --input categories.json \
  --output relationships.json \
  --min_strength 0.5
```

查看 `summary.total_relations`：
- ≥10：密度良好
- 5-10：基本可以
- <5：密度不足

**人工检查**：
- 绘制关系网络图
- 检查是否有孤立的范畴
- 询问"范畴间的联系是否充分？"

### 案例

**密度良好的关系网络**：
```
学习方法差异 --[因果,0.85]--> 学业适应困难
学业适应困难 --[策略,0.78]--> 寻求学业支持
寻求学业支持 --[策略,0.82]--> 成绩提升
成绩提升 --[互动,0.75]--> 学习自信心
学习自信心 --[互动,0.70]--> 学习投入
学习投入 --[策略,0.68]--> 成绩提升  // 反馈循环
个人性格 --[条件,0.65]--> 寻求学业支持  // 中介条件
社会支持 --[条件,0.72]--> 寻求学业支持  // 中介条件
```

**总关系数**：12个
**关系类型**：因果、策略、互动、条件
**复杂性**：包含反馈循环和中介条件

## 3. 理论的变异性（Variation）

### 标准

**考虑了不同情境下的变化**

✅ **检查清单**：
- [ ] 识别了情境差异（语境）
- [ ] 分析了情境如何影响现象
- [ ] 考虑了不同情境下的策略差异

**识别了中介条件的影响**

✅ **检查清单**：
- [ ] 识别了促进因素
- [ ] 识别了限制因素
- [ ] 分析了条件如何改变策略效果

**分析了策略的多样性**

✅ **检查清单**：
- [ ] 识别了多种策略
- [ ] 分析了策略的适用条件
- [ ] 考虑了策略组合

### 评估方法

**人工检查**：
- 对比不同受访者的经历
- 识别成功案例和失败案例的差异
- 询问"为什么有的人成功，有的人失败？"

### 案例

**变异性良好的分析**：
```
策略：寻求学业支持

情境变化：
- 重点大学（竞争激烈）：更多学生寻求帮助
- 普通大学（竞争缓和）：较少学生寻求帮助

中介条件影响：
- 外向性格：更主动、更频繁地寻求帮助
- 内向性格：较少寻求帮助，倾向自学
- 良好社会支持：寻求帮助更有效
- 缺乏社会支持：寻求帮助效果有限

策略多样性：
- 策略A：向教师请教（直接、正式）
- 策略B：参加学习小组（间接、同伴）
- 策略C：寻找学长指导（间接、非正式）
```

## 4. 理论的整合性（Integration）

### 标准

**核心现象明确**

✅ **检查清单**：
- [ ] 核心现象清晰
- [ ] 核心现象有理论意义
- [ ] 核心现象能概括研究主题

**所有范畴都与核心现象相关**

✅ **检查清单**：
- [ ] 每个范畴都能连接到核心现象
- [ ] 没有孤立的范畴
- [ ] 关系网络围绕核心现象展开

**形成了连贯的理论故事**

✅ **检查清单**：
- [ ] 能用一段话讲清楚理论
- [ ] 逻辑流畅、因果清晰
- [ ] 听众能够理解

### 评估方法

**理论故事测试**：
用一段话（200字）讲述理论：
```
大学新生由于学习方法差异和课程难度提升，经历了学业适应困难（现象）。
在重点大学的竞争环境下（语境），学生采取了寻求学业支持、调整学习方法等
策略（行动）。然而，个人性格和社会支持网络（中介条件）影响了策略的效果。
最终，成功适应的学生不仅成绩提升，还建立了有效的学习方法和社交网络（结果）。
```

**检查**：
- [ ] 包含了所有六个组件
- [ ] 逻辑清晰、流畅
- [ ] 能够被非专业人士理解

### 案例

**整合性良好**：
- 核心现象明确："学业适应困难"
- 所有范畴都相关（寻求支持、调整方法、建立网络都是应对策略）
- 能够讲清楚理论故事

**整合性不足**：
- 核心现象不明确（是"学业困难"还是"心理压力"？）
- 有孤立的范畴（"恋爱关系"与学业适应关系不明确）
- 理论故事不连贯

## 综合评估

### 评分表

| 维度 | 权重 | 分数 | 加权分 |
|-----|------|------|--------|
| 完整性 | 30% | 0-100 | |
| 密度 | 25% | 0-100 | |
| 变异性 | 20% | 0-100 | |
| 整合性 | 25% | 0-100 | |
| **总分** | 100% | | 0-100 |

**评分标准**：
- 90-100：优秀，理论质量高
- 80-89：良好，可以发表
- 70-79：合格，需要改进
- <70：不合格，需要大幅修改

### 使用工具评估

```bash
python scripts/assess_paradigm_quality.py \
  --input paradigm.json \
  --output quality_report.json
```

**输出示例**：
```json
{
  "overall_score": 85,
  "dimensions": {
    "completeness": {"score": 90, "issues": []},
    "density": {"score": 82, "issues": ["关系数量略少"]},
    "variation": {"score": 80, "issues": ["中介条件分析可以更深入"]},
    "integration": {"score": 88, "issues": []}
  },
  "recommendation": "良好，可以发表。建议增加关系分析深度。"
}
```

## 改进建议

### 完整性不足
- **问题**：缺少某些组件
- **解决**：回到数据，寻找缺失的组件
- **参考**：`../troubleshooting/paradigm-issues.md`

### 密度不足
- **问题**：关系太少、太简单
- **解决**：深入分析范畴间的关系，识别更多关系
- **参考**：`../relationship-types/INDEX.md`

### 变异性不足
- **问题**：忽略了情境差异和中介条件
- **解决**：对比不同案例，识别差异和条件
- **参考**：`six-components.md` - 中介条件章节

### 整合性不足
- **问题**：核心现象不明确，理论故事不连贯
- **解决**：重新审视核心现象，撰写理论故事
- **参考**：`../category-construction/step3-naming.md`
