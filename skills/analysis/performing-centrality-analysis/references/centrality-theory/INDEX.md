# 中心性理论指南

> **渐进式学习**：本文档提供四种中心性的理论基础。

## 四种中心性对比

| 中心性 | 公式 | 计算复杂度 | 适用场景 |
|-------|------|-----------|---------|
| 度中心性 | C_D(v) = deg(v)/(n-1) | O(n) | 识别活跃节点 |
| 接近中心性 | C_C(v) = (n-1)/Σd(v,u) | O(n²) | 识别信息传播快的节点 |
| 介数中心性 | C_B(v) = Σσ_st(v)/σ_st | O(n³) | 识别桥梁节点 |
| 特征向量中心性 | Ax = λx | O(n²) | 识别有影响力的节点 |

---

## 1. 度中心性（Degree Centrality）

**定义**：节点的直接连接数量

**公式**：
```
C_D(v) = deg(v) / (n-1)
```
- deg(v)：节点v的度数
- n：网络节点总数

**解释**：
- 值域：[0, 1]
- 高值：社交活跃，直接连接多
- 低值：边缘节点，连接少

**应用**：
- 识别社交网络中的活跃分子
- 找出信息发布的源头

---

## 2. 接近中心性（Closeness Centrality）

**定义**：节点到达其他节点的平均距离的倒数

**公式**：
```
C_C(v) = (n-1) / Σ d(v,u)
```
- d(v,u)：节点v到u的最短路径长度

**解释**：
- 值域：(0, 1]
- 高值：能快速到达网络中的所有节点
- 低值：远离网络中心

**应用**：
- 识别信息传播效率高的节点
- 找出物流网络中的最佳位置

---

## 3. 介数中心性（Betweenness Centrality）

**定义**：节点在最短路径中出现的频率

**公式**：
```
C_B(v) = Σ σ_st(v) / σ_st
```
- σ_st：节点s到t的最短路径数
- σ_st(v)：经过节点v的最短路径数

**解释**：
- 值域：[0, 1]
- 高值：控制信息流动的桥梁
- 低值：不在关键路径上

**应用**：
- 识别网络中的桥梁和中介
- 找出信息传播的瓶颈

**算法**：Brandes算法（O(nm)，m为边数）

---

## 4. 特征向量中心性（Eigenvector Centrality）

**定义**：节点连接到其他重要节点的程度

**公式**：
```
Ax = λx
```
- A：邻接矩阵
- x：特征向量（中心性值）
- λ：最大特征值

**解释**：
- 值域：[0, 1]（标准化后）
- 高值：连接到其他重要节点
- 低值：连接到不重要节点

**应用**：
- 识别具有间接影响力的节点
- 找出意见领袖

---

## 选择指南

**研究问题**决定使用哪种中心性：

- **谁最活跃？** → 度中心性
- **谁传播信息最快？** → 接近中心性
- **谁是桥梁？** → 介数中心性
- **谁最有影响力？** → 特征向量中心性

**综合分析**：同时计算多种中心性，找出在多个维度上都突出的节点
