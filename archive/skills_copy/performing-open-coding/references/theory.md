# 开放编码理论基础

> **渐进式披露**：本文档提供开放编码的理论背景。详细案例见 `examples.md`，故障排除见 `troubleshooting.md`。

## 核心概念

### 什么是开放编码

开放编码是扎根理论的第一阶段，通过逐行分析质性数据，识别和命名概念。

**关键特征**：
- **行动导向**：概念命名使用动词（如"寻求帮助"而非"帮助"）
- **持续比较**：不断比较新旧数据，精炼概念
- **理论敏感性**：保持开放态度，让理论从数据中涌现

### 理论流派

**Glaser派（经典扎根理论）**：
- 强调理论的自然涌现
- 避免预设理论框架
- 编码更加开放和灵活

**Strauss派（程序化扎根理论）**：
- 提供系统化的编码程序
- 使用编码范式（Paradigm）
- 本技能主要遵循此流派

**Charmaz派（建构主义扎根理论）**：
- 强调研究者的主体性
- 重视社会建构过程

## 核心原则

### 1. 概念命名原则

**行动导向命名**：
```
✅ 好的命名：
- "寻求老师帮助"（动词+对象）
- "建立同伴关系"（动词+结果）
- "适应大学生活"（动词+情境）

❌ 避免的命名：
- "帮助"（名词，过于抽象）
- "老师"（主体，非行动）
- "大学"（情境，非概念）
```

### 2. 抽象层次

**三个层次**：
1. **描述性编码**：贴近原始数据（"向老师询问问题"）
2. **概念性编码**：适度抽象（"寻求学术支持"）
3. **理论性编码**：高度抽象（"资源动员策略"）

**本技能关注**：概念性编码（第2层）

### 3. 持续比较方法

**比较类型**：
- **事件与事件**：比较不同访谈中的相似现象
- **概念与概念**：比较已有概念的异同
- **概念与数据**：用新数据验证已有概念

**操作要点**：
- 每识别一个新概念，立即与已有概念比较
- 相似度高（>0.8）→ 考虑合并
- 相似度中（0.5-0.8）→ 建立关联
- 相似度低（<0.5）→ 独立概念

## 中文语境适配

### 文化特殊性

**关系导向**：
- 中文研究中"关系"概念的重要性
- 示例："建立师生关系"、"维护同学关系"

**集体主义**：
- 强调群体和谐
- 示例："融入集体"、"适应群体规范"

**面子文化**：
- 重视社会评价
- 示例："维护面子"、"避免尴尬"

### 语言特点

**词汇结构**：
- 中文多为双音节词（如"学习"、"帮助"）
- 四字结构常见（如"寻求帮助"、"建立关系"）

**语义丰富性**：
- 同一概念可能有多种表达
- 需要识别同义词和近义词
- 工具支持：使用jieba分词 + 语义相似度

## 质量标准

### 优秀编码的特征

1. **命名清晰**：一目了然，无歧义
2. **定义完整**：说明概念的内涵和外延
3. **示例充分**：至少2个具体例子
4. **抽象适度**：既不过于具体，也不过于抽象
5. **理论相关**：与研究问题相关

### 评分标准

- **90-100分（A）**：命名、定义、示例全部优秀
- **80-89分（B）**：大部分符合标准，少量问题
- **70-79分（C）**：部分符合标准，需改进
- **60-69分（D）**：勉强及格，问题较多
- **<60分（F）**：不符合标准，需重新编码

## 常见挑战

### 挑战1：概念过于具体

**表现**：
- "向张老师询问微积分问题"（过于具体）

**解决**：
- 提升抽象层次："寻求学术支持"

### 挑战2：概念过于抽象

**表现**：
- "社会互动"（过于抽象）

**解决**：
- 降低抽象层次："建立同学关系"

### 挑战3：重复编码

**表现**：
- "寻求帮助"、"寻求支持"、"寻求援助"（重复）

**解决**：
- 使用工具识别相似度，合并为统一概念

## 工具支持

### 自动化工具

本技能提供5个脚本工具：

1. **preprocess_text.py**：文本预处理
   - 中文分词（jieba）
   - 停用词过滤
   - 语义分段

2. **auto_loader.py**：快速概念提取
   - 自动识别高频词
   - 概念类型分类
   - 编码建议生成

3. **compare_codes.py**：持续比较
   - 相似度计算
   - 重复识别
   - 合并建议

4. **cluster_concepts.py**：概念聚类
   - K-means聚类
   - 层次聚类
   - 聚类可视化

5. **validate_codes.py**：编码验证
   - 命名规范检查
   - 定义质量检查
   - 示例充分性检查

### 使用建议

**推荐流程**：
```bash
# 1. 预处理文本
python scripts/preprocess_text.py --input interview.txt --output clean.json

# 2. 快速概念提取
python scripts/auto_loader.py --input interview.txt --output concepts.json

# 3. 持续比较
python scripts/compare_codes.py --input concepts.json --output comparison.json

# 4. 概念聚类（可选）
python scripts/cluster_concepts.py --input concepts.json --output clusters.json

# 5. 编码验证
python scripts/validate_codes.py --input concepts.json --output validation.json
```
