# 范畴构建实践指南

> **渐进式学习**：本文档提供范畴构建概览。详细步骤见子文档。

## 什么是范畴构建

范畴构建是轴心编码的核心任务，将开放编码产生的零散概念整合为系统性的范畴体系。

**输入**：开放编码的概念列表（通常20-50个）

**输出**：系统化的范畴体系（通常5-10个范畴）

**过程**：聚类 → 精炼 → 命名 → 验证

---

## 快速开始

### 工具链

```bash
# 第一步：概念聚类
python scripts/identify_categories.py \
  --input codes.json \
  --output categories_v1.json \
  --n_categories 5

# 第二步：质量验证
python scripts/validate_categories.py \
  --input categories_v1.json \
  --output validation.json
```

### 人工精炼

工具提供初步聚类，人工进行：
1. 检查概念归类的合理性
2. 完善范畴命名（行动导向）
3. 撰写清晰的范畴定义
4. 提供典型案例支持

---

## 四个关键步骤

### 第一步：概念聚类
**目标**：将相似的概念归为一组

**方法**：
- 使用工具自动聚类（K-means、层次聚类）
- 人工检查和调整

**输出**：初步的范畴分组

**详见**：`step1-clustering.md`

---

### 第二步：范畴精炼
**目标**：完善范畴的定义和内容

**任务**：
- 撰写范畴定义
- 提供典型案例
- 识别属性维度
- 建立范畴层级

**输出**：精炼的范畴体系

**详见**：`step2-refining.md`

---

### 第三步：范畴命名
**目标**：为范畴选择恰当的名称

**原则**：
- 行动导向（使用动词）
- 适度抽象（不过具体、不过抽象）
- 理论意义（体现理论概念）

**输出**：优化的范畴名称

**详见**：`step3-naming.md`

---

### 第四步：质量验证
**目标**：确保范畴构建的质量

**检查**：
- 内部一致性（范畴内概念相关）
- 外部区分度（范畴间明确区分）
- 数据支持（充分的证据）
- 理论意义（有理论价值）

**输出**：质量报告

**详见**：`step2-refining.md` - 验证章节

**内部一致性**：
- [ ] 范畴内的概念高度相关
- [ ] 范畴定义清晰完整

**外部区分度**：
- [ ] 范畴间有明确区分

**数据支持**：
- [ ] 每个范畴至少3个概念
- [ ] 每个范畴至少20个数据片段

**理论意义**：
- [ ] 范畴有理论价值
