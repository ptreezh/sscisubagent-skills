# 理论饱和度检验指南

> **渐进式学习**：本文档提供饱和度检验的概览和标准。

## 什么是理论饱和度

理论饱和度（Theoretical Saturation）指理论发展到一定程度，新数据不再产生新的概念、范畴或关系，理论框架已经完整和稳定。

**Glaser & Strauss (1967) 的定义**：
> "当研究者发现新数据不再带来新的理论洞察时，理论达到饱和。"

---

## 四个检验维度

### 1. 概念饱和度
**标准**：新概念出现率 < 5%

**检验方法**：
```bash
python scripts/check_saturation.py \
  --existing existing_concepts.json \
  --new new_data.json
# 查看 concept_saturation.new_concept_rate
```

**判断**：
- < 5%：✅ 已饱和
- 5-10%：⚠️ 接近饱和
- > 10%：❌ 未饱和

---

### 2. 范畴完整性
**标准**：范畴完整率 ≥ 90%

**检查项**：
- 范畴有清晰定义
- 范畴有属性维度
- 范畴有典型案例

**判断**：
- ≥ 90%：✅ 已饱和
- 70-90%：⚠️ 基本完整
- < 70%：❌ 不完整

---

### 3. 关系稳定性
**标准**：新关系出现率 < 10%

**检验方法**：
- 对比新旧数据的关系网络
- 计算新关系的比例

**判断**：
- < 10%：✅ 已饱和
- 10-20%：⚠️ 接近饱和
- > 20%：❌ 未饱和

---

### 4. 理论完整性
**标准**：理论完整率 ≥ 90%

**检查项**：
- 有明确的核心范畴
- 有系统的理论命题
- 有完整的概念框架
- 有清晰的理论边界

**判断**：
- ≥ 90%：✅ 已饱和
- 70-90%：⚠️ 基本完整
- < 70%：❌ 不完整

---

## 综合判断

**完全饱和**：所有4个维度都达到饱和标准

**部分饱和**：3个维度达到饱和

**未饱和**：少于3个维度达到饱和

---

## 使用工具检验

```bash
python scripts/check_saturation.py \
  --existing theory.json \
  --new new_data.json \
  --output saturation_report.json
```

**输出示例**：
```json
{
  "summary": {
    "saturation_score": 0.92,
    "is_fully_saturated": true,
    "saturated_dimensions": 4
  }
}
```

## 饱和度不足的处理

### 概念未饱和
- 继续收集数据
- 针对性访谈关键信息人
- 理论抽样

### 范畴不完整
- 补充范畴定义
- 分析属性维度
- 提供更多案例

### 关系不稳定
- 深入分析范畴关系
- 验证关系的证据
- 补充关系数据

### 理论不完整
- 完善理论命题
- 整合概念框架
- 明确理论边界

---

## 质量标准

**Strauss & Corbin (1998) 的标准**：
- 理论能够解释大部分数据
- 理论具有足够的密度和复杂性
- 理论考虑了情境变异
- 理论形成了连贯的整体
