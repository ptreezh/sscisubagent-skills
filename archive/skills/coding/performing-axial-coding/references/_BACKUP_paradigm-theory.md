# Paradigm模型理论基础

> **渐进式披露**：本文档提供Paradigm模型的核心理论。完整案例见`category-examples.md`，关系类型详解见`relationship-types.md`。

## 核心概念

### 什么是Paradigm模型

Paradigm（范式）模型是Strauss & Corbin（1990, 1998）提出的轴心编码核心工具，用于系统化地建立范畴间的逻辑关系。它将复杂的社会现象分解为六个组件，形成"条件→现象→行动→结果"的因果链条。

**核心价值**：
- 将零散的范畴整合为系统性的理论框架
- 揭示社会现象的内在逻辑和机制
- 为理论构建提供清晰的分析路径

### 理论流派

**Strauss & Corbin流派**（本技能采用）：
- 强调结构化的编码程序
- 使用Paradigm模型作为分析工具
- 适合初学者和需要系统性框架的研究

**Glaser流派**：
- 反对预设的编码框架
- 强调理论的"自然涌现"
- 适合经验丰富的研究者

**Charmaz建构主义流派**：
- 强调研究者的主观建构
- 关注情境和视角的多样性
- 适合解释主义研究

## Paradigm的六个组件

### 1. 现象（Phenomenon）

**定义**：研究的核心事件、发生的事情、行动/互动所指向的中心议题。

**识别方法**：
- 寻找访谈中反复出现的主题
- 识别参与者最关注的问题
- 找到连接最多其他范畴的核心范畴

**案例**（大学生学业适应研究）：
```
现象：学业适应困难
定义：新生在大学学习过程中遇到的学习方法、课程难度、时间管理等方面的挑战
证据：访谈中80%的学生提到"不适应"、"困难"、"压力大"
```

**工具支持**：
```bash
# 使用identify_categories.py识别核心范畴
python scripts/identify_categories.py --input codes.json --output categories.json
# 查看summary.core_categories字段
```

### 2. 因果条件（Causal Conditions）

**定义**：导致现象发生的事件、情境或背景因素。

**识别方法**：
- 寻找"导致"、"引起"、"因为"等因果关键词
- 分析时间序列（什么在现象之前发生）
- 询问"为什么会出现这个现象？"

**案例**：
```
因果条件1：中学与大学学习方式差异
- 中学：被动接受、教师主导、题海战术
- 大学：主动学习、自主管理、理解为主

因果条件2：课程难度突然提升
- 专业课抽象程度高
- 知识量大、进度快
- 缺乏基础知识衔接

因果条件3：时间管理能力不足
- 缺乏自主规划经验
- 课外活动多、诱惑大
- 拖延习惯未改变
```

**工具支持**：
```bash
# 使用build_relationships.py识别因果关系
python scripts/build_relationships.py --input categories.json --output relationships.json
# 查看details.causal_relations字段
```

### 3. 语境（Context）

**定义**：现象发生的特定情境、背景条件，回答"在什么条件下"。

**与因果条件的区别**：
- 因果条件：直接导致现象（"为什么"）
- 语境：现象发生的环境（"在哪里"、"什么时候"）

**案例**：
```
语境1：重点大学的学业竞争环境
- 同学普遍学习能力强
- 课程要求高、淘汰率高
- 绩点竞争激烈

语境2：远离家乡的独立生活
- 缺少家长监督和支持
- 需要自己处理生活琐事
- 孤独感和思乡情绪

语境3：大一新生的特殊阶段
- 身份转换期（中学生→大学生）
- 社交网络重建期
- 职业规划迷茫期
```

### 4. 中介条件（Intervening Conditions）

**定义**：促进或限制行动策略的更广泛的结构性条件。

**识别方法**：
- 寻找"但是"、"然而"、"虽然"等转折词
- 分析为什么同样的行动在不同情况下效果不同
- 识别资源、权力、文化等结构性因素

**案例**：
```
中介条件1：个人性格特质
- 内向学生难以主动求助
- 完美主义者容易焦虑
- 乐观者更易适应

中介条件2：社会支持网络
- 有朋友支持的学生适应更快
- 师生关系质量影响求助行为
- 家庭支持提供情感缓冲

中介条件3：学校资源可及性
- 学业辅导中心的开放时间
- 教师答疑的便利程度
- 心理咨询服务的可获得性
```

### 5. 行动/互动策略（Action/Interaction Strategies）

**定义**：人们为应对现象而采取的有目的、有意识的行动。

**识别方法**：
- 寻找动词短语（"寻求"、"调整"、"建立"）
- 询问"人们做了什么来应对？"
- 分析成功和失败的策略

**案例**：
```
策略1：主动寻求学业支持
- 向教师请教问题
- 参加学习小组
- 寻找学长学姐指导

策略2：调整学习方法
- 从题海战术转向理解为主
- 制定学习计划
- 使用时间管理工具

策略3：建立社会支持网络
- 结交志同道合的朋友
- 加入学习型社团
- 保持与家人的沟通

策略4：心态调整
- 接受"不完美"的自己
- 降低不合理期望
- 培养积极归因方式
```

**工具支持**：
```bash
# 使用construct_paradigm.py识别行动策略
python scripts/construct_paradigm.py --input relationships.json --output paradigm.json
# 查看details.paradigm.action_strategies字段
```

### 6. 结果（Consequences）

**定义**：行动/互动策略产生的结果，可以是预期的或非预期的。

**识别方法**：
- 寻找"结果"、"导致"、"最终"等关键词
- 分析时间序列（什么在行动之后发生）
- 区分短期结果和长期结果

**案例**：
```
积极结果：
- 学业成绩提升
- 学习自信心增强
- 建立了有效的学习方法
- 形成了良好的社交网络

消极结果：
- 部分学生过度依赖他人帮助
- 时间管理仍然困难
- 焦虑情绪未完全缓解

意外结果：
- 发现了新的兴趣方向
- 建立了深厚的友谊
- 学会了自我反思和成长
```

## Paradigm模型的构建流程

### 第一步：识别核心现象

**使用工具**：
```bash
python scripts/identify_categories.py --input codes.json --output categories.json
```

**人工判断**：
- 选择最能概括研究主题的范畴
- 确保现象具有足够的理论意义
- 验证现象在数据中有充分支持

### 第二步：识别因果条件和语境

**使用工具**：
```bash
python scripts/build_relationships.py --input categories.json --output relationships.json
```

**人工分析**：
- 区分"为什么"（因果）和"在哪里"（语境）
- 寻找多重因果条件
- 分析条件的相对重要性

### 第三步：识别行动策略

**人工分析**：
- 列出所有应对策略
- 分析策略的有效性
- 识别策略的变化和发展

### 第四步：识别中介条件

**人工分析**：
- 分析为什么同样的策略效果不同
- 识别结构性限制因素
- 考虑个人、社会、文化层面的条件

### 第五步：识别结果

**人工分析**：
- 区分短期和长期结果
- 识别预期和非预期结果
- 分析结果的理论意义

### 第六步：整合Paradigm模型

**使用工具**：
```bash
python scripts/construct_paradigm.py --input relationships.json --output paradigm.json
```

**验证模型**：
- 检查逻辑链条的完整性
- 验证每个组件的数据支持
- 评估模型的解释力

## 中国本土化适配

### 陈向明的实践（2000）

陈向明在《质的研究方法与社会科学研究》中强调：

**关系导向的Paradigm**：
- 因果条件：不仅关注个人因素，更要分析关系网络
- 行动策略：重视"关系"的建立和维护
- 中介条件：考虑"面子"、"人情"等文化因素

**集体主义的影响**：
- 现象：往往涉及个人与集体的关系
- 策略：强调集体行动和相互支持
- 结果：评估对集体的影响

### 本土化要点

**1. 关系网络分析**：
- 将"关系"作为独立的分析维度
- 分析关系的质量和类型
- 考虑关系的动态变化

**2. 权威尊重**：
- 分析权威（教师、领导）的影响
- 考虑等级结构的作用
- 识别自上而下的策略

**3. 面子文化**：
- 分析"面子"如何影响行动选择
- 考虑"丢面子"的担忧
- 识别间接沟通策略

**4. 情境敏感性**：
- 中国人的行为高度依赖情境
- 语境和中介条件尤为重要
- 分析"变通"策略

## 质量标准

### Strauss & Corbin (1998, pp.269-272) 的质量标准

**理论的完整性**：
- ✅ 所有六个组件都有充分的数据支持
- ✅ 组件间的逻辑关系清晰
- ✅ 模型能够解释大部分数据

**理论的密度**：
- ✅ 范畴之间有丰富的关系
- ✅ 关系有充分的证据支持
- ✅ 理论有足够的复杂性

**理论的变异性**：
- ✅ 考虑了不同情境下的变化
- ✅ 识别了中介条件的影响
- ✅ 分析了策略的多样性

**理论的整合性**：
- ✅ 核心现象明确
- ✅ 所有范畴都与核心现象相关
- ✅ 形成了连贯的理论故事

## 工具支持说明

本技能提供4个核心脚本辅助Paradigm构建：

**1. identify_categories.py**：
- 自动聚类概念为范畴
- 识别核心范畴
- 建立范畴层级

**2. build_relationships.py**：
- 识别因果关系
- 识别条件关系
- 识别策略关系
- 识别互动关系

**3. construct_paradigm.py**：
- 自动构建Paradigm模型
- 验证模型完整性
- 生成质量评估报告

**4. analyze_properties.py**（可选）：
- 识别范畴的属性
- 定义属性的维度
- 分析案例在维度上的分布

**注意**：工具提供初步分析，最终判断需要研究者的定性分析和理论敏感性。
