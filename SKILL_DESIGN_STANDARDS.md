# 技能设计标准 - 主控文档

## 目标
确保所有技能设计对齐 agentskills.io 标准，提供精心、全面、系统的思考和设计。

## 核心原则

### 1. 深入的理论基础
- 每个技能必须基于坚实的学术理论
- 提供相关理论流派的详细说明
- 阐述理论在实际应用中的意义和限制

### 2. 详尽的实践指导
- 提供逐步的操作指南
- 包含最佳实践和常见陷阱
- 提供质量控制和验证机制

### 3. 全面的资源支持
- 提供丰富的参考资料
- 包含实际案例和示例
- 提供故障排除指南

### 4. 完整的工具链
- 提供自动化脚本（如适用）
- 包含测试和验证工具
- 提供输出格式规范

## 技能结构要求

### A. 基础元数据
```yaml
name: 技能名称
description: 技能功能的简洁描述
version: 版本号 (遵循语义化版本)
author: 作者信息
tags: [相关标签]
```

### B. 技能文档 (SKILL.md)
必须包含以下部分：

#### 1. 概述 (Overview)
- 技能的目的和意义
- 适用场景和用户群体
- 与其他技能的关系

#### 2. 使用时机 (When to Use)
- 明确的使用条件
- 触发技能的关键词或场景
- 不适用的情况

#### 3. 快速开始 (Quick Start)
- 最简单的使用方法
- 3-5个核心步骤
- 必需的前置条件

#### 4. 核心流程 (Core Process)
- 详细的分步指导
- 每个步骤的目标和方法
- 关键决策点和判断标准

#### 5. 输出格式 (Output Format)
- 标准化输出格式
- 元数据要求
- 验证机制

#### 6. 质量标准 (Quality Standards)
- 质量检查清单
- 验证指标
- 常见错误和解决方案

#### 7. 深入学习 (Advanced Learning)
- 理论基础文档
- 实践案例
- 故障排除指南

#### 8. 完成标志 (Completion Criteria)
- 明确的完成标准
- 交付物清单
- 质量验收标准

### C. 脚本目录 (scripts/)
- 自动化处理脚本
- 输入验证脚本
- 输出格式化脚本
- 测试和验证脚本

### D. 参考资料目录 (references/)
- 理论文档
- 实践案例
- 故障排除指南
- 中文语境特殊性说明

### E. 测试目录 (tests/)
- 单元测试
- 集成测试
- 端到端测试
- 性能基准测试

### F. 项目配置文件
- pyproject.toml 或 package.json
- 依赖管理
- 构建脚本

## 质量评估标准

### 1. 完整性 (Completeness) - 30分
- [ ] 所有必需部分都已包含 (10分)
- [ ] 每个部分都有实质性内容 (10分)
- [ ] 提供了充分的示例和案例 (10分)

### 2. 深度 (Depth) - 25分
- [ ] 理论基础充分 (10分)
- [ ] 实践指导详细 (10分)
- [ ] 提供了深入的学习资源 (5分)

### 3. 实用性 (Practicality) - 25分
- [ ] 提供了自动化工具 (10分)
- [ ] 操作步骤清晰可行 (10分)
- [ ] 包含质量控制机制 (5分)

### 4. 一致性 (Consistency) - 10分
- [ ] 遵循统一的结构标准 (5分)
- [ ] 术语使用一致 (5分)

### 5. 可维护性 (Maintainability) - 10分
- [ ] 包含测试用例 (5分)
- [ ] 提供了清晰的文档 (5分)

## 实施计划

### 阶段1: 评估 (已完成)
- 评估现有技能的质量
- 识别需要改进的技能

### 阶段2: 标准化 (当前阶段)
- 制定技能设计标准
- 创建模板和工具

### 阶段3: 重构 (待执行)
- 按标准重构现有技能
- 创建新技能

### 阶段4: 验证 (待执行)
- 测试重构后的技能
- 收集反馈并改进

## 模板结构

为确保所有技能都遵循此标准，将创建以下模板：
- SKILL.md 模板
- 脚本目录结构模板
- 参考资料目录结构模板
- 测试目录结构模板
- 项目配置文件模板

此标准将作为所有技能开发和重构的基准，确保每个技能都达到 agentskills.io 的质量要求。

详细的实施指南、最佳实践和评估流程请参考《SKILL_DESIGN_GUIDELINES.md》。
