# fsQCA理论分析详纲

## 详细分析流程

### 第一步：研究问题澄清
1. 明确研究问题和目标
2. 确定结果变量(Y)
3. 识别潜在的前因条件(X1, X2, ..., Xn)
4. 确定案例集合

### 第二步：理论框架构建
1. 基于现有理论确定条件变量
   - 核心条件：理论上对结果有直接影响的变量
   - 边界条件：影响条件与结果关系的调节变量
   - 控制条件：需要控制的潜在混淆变量

2. 构建因果机制假设
   - 识别可能的因果路径
   - 考虑条件间的相互作用
   - 预期的组态效应

### 第三步：概念操作化
1. 将抽象概念转化为可测量指标
2. 确定每个条件的理论含义
3. 定义条件的极值和中间状态

### 第四步：校准策略规划
1. 为每个条件确定校准方法
   - 直接校准：基于理论锚点和经验数据
   - 间接校准：基于分类到隶属度的映射
   - 阈值校准：基于特定阈值的二分或多分

2. 确定校准的理论依据
   - 基于理论的锚点设定
   - 基于经验数据的验证

## 详细输出要求

### 理论框架文档
- 研究问题陈述
- 理论模型图示
- 条件变量定义表
- 因果机制假设

### 条件选择理由
- 每个条件的理论依据
- 条件间关系的理论解释
- 排除某些条件的理由

### 预期结果
- 预期的因果组态类型
- 不同组态的理论含义
- 可能的边缘案例考虑

## 详细质量检查清单
- [ ] 理论框架是否完整且逻辑一致？
- [ ] 条件选择是否基于充分的理论依据？
- [ ] 是否考虑了条件间的相互作用？
- [ ] 校准策略是否合理且可行？
- [ ] 因果机制假设是否清晰明确？

## 详细注意事项
1. fsQCA处理连续隶属度，需要特别关注条件的模糊性特征
2. 强调组态思维而非变量思维
3. 考虑因果的非对称性（充分性vs必要性）
4. 保持案例导向的研究视角

## 专业术语详解
- 模糊集：允许元素以不同程度隶属于集合
- 隶属度：元素隶属于集合的程度（0到1）
- 组态：条件的特定组合
- 一致性：条件组合对结果的充分性程度
- 覆盖度：条件组合对结果的解释程度
- 必要性：条件对结果的必要程度
- 逻辑余项：真值表中未观察到的配置
